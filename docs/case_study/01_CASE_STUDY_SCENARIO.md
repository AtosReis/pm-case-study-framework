# APPLIED CASE STUDY — COMPLIANCEBRIEF x PARTNERCRM

## Scenario

ComplianceBrief is a fictional B2B company that produces weekly compliance research content.

PartnerCRM is a fictional larger CRM platform used by sales and customer-success teams.

ComplianceBrief wants to explore a partnership where PartnerCRM users can generate newsletters using ComplianceBrief content without leaving the PartnerCRM workflow.

## Ambiguous Client Request

We want to integrate our newsletter product with a larger partner CRM so their users can create newsletters using our content.

## PM Interpretation

This is not only an AI generation problem.

It is a product integration problem involving:

- authentication;
- content access;
- permissions;
- branding;
- newsletter creation;
- recipient selection;
- approval;
- distribution;
- analytics;
- data ownership.

## MVP Goal

Allow a PartnerCRM user to access approved ComplianceBrief content, generate a newsletter draft, review it, select recipients, and prepare it for distribution.

## MVP Non-Goals

- No real SSO implementation.
- No real CRM integration.
- No real email sending.
- No real LLM API.
- No production authentication.
- No real customer data.
- No compliance/legal guarantee.

## Learning Goal

Use this scenario to demonstrate how a PM should move from ambiguity to engineering-ready product work.
